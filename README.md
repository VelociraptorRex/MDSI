# MDSI — Español esencial

Содержательная версия 1.10 системы минимально достаточного изучения языка. Пакет `es-419` построен как спокойная связанная энциклопедия: 500 лексических единиц, 25 коротких грамматических тем в 8 разделах, 32 паттерна, 8 диалогов, банк из 2 187 упражнений, генератор заданий на спряжение и готовые карточки Anki, Quizlet и Полиглот используют общую модель данных и общие компоненты.

Практика поддерживает счёт по каждому уровню, выбор от 10 до 100 заданий на узнавание, управление ответами клавишами 1–4, автоматический переход после правильного ответа, сборку фразы из слов и ручной ввод, отдельный тренажёр спряжений и честную самопроверку свободных ситуационных реплик. В тренажёре настраиваются сложность, число пропусков, от 5 до 100 глаголов, пул из 20 временных форм, тип спряжения и представление карточек сеткой либо веерной каруселью. Карусель показывает обе соседние карточки, поддерживает клик, горизонтальный жест, экранные кнопки и клавиши ←/→; после безошибочной карточки через 1,2 секунды открывается следующая незаполненная, а ошибка останавливает автопереход. Каждую парадигму можно озвучить вручную, полностью правильная карточка произносится автоматически. После последнего ответа итог появляется сразу; из него можно начать обычный новый набор или повторить только глаголы, в которых была допущена ошибка. На первых трёх уровнях правильный испанский ответ однократно озвучивается предпочтительно женским испанским голосом браузера; общий переключатель позволяет мгновенно отключить и снова включить звук.

Лексикон хранит полную модель спряжения для всех 128 глаголов. Переключатель над таблицей выбирает компактное учебное ядро или все 20 форм Indicativo, Subjuntivo и Imperativo. Каждое время показано отдельной мини-таблицей с местоимениями слева. Карточка открывается временно по наведению или закрепляется щелчком; целевая форма во всех 500 естественных примерах выделяется красным, а отдельная кнопка озвучивает лемму.

## Запуск

```bash
npm install
npm run dev
```

Производственная сборка:

```bash
npm run build
```

Готовые статические страницы появляются в `dist/`.

В GitHub Actions базовый путь определяется автоматически из имени репозитория: проект `MDSI` публикуется по адресу `/MDSI/`, а локальная и обычная производственная сборки продолжают работать от корня.

Anki и Quizlet пересобираются из лексикона, банка фраз и паттернов:

```bash
npm run generate:cards
```

Результаты `.apkg` для Anki и `.txt` с табуляцией для Quizlet сохраняются в `public/downloads/`. Таблица Полиглота `.xlsx` строится из тех же 1 044 нормализованных строк: семь совместимых колонок, без обратных дублей, с полностью пустым столбцом «Выучено». Все три файла автоматически попадают в статическую сборку сайта.

## Локальный пакет для macOS

```bash
npm run package:local
```

Команда создаёт папку `local-package/MDSI-local` с готовой сборкой и файлом `Открыть MDSI.command`. Он запускает локальный сервер и открывает сайт в браузере. Непосредственное открытие HTML через `file://` не поддерживается браузерами для приложений с JavaScript-модулями.

## Данные

Актуальный исходный файл лексического ядра находится в `content/es-419/lexico_es_500_v02.xlsx`. Команда `npm run import:lexicon` преобразует его в машинный JSON для интерфейса и проверяет наличие ровно 500 строк. В v0.2 отдельно добавлены базовые вопросительные, неопределённые и отрицательные слова; менее необходимые описательные единицы удалены, поэтому объём остался прежним.

## Архитектура

- `src/components/` — общие компоненты: шапка, поиск, содержание, popover, таблицы спряжения и тренажёр;
- `src/assets/es.svg` — флаг Испании в шапке; файл можно заменить собственным SVG с тем же именем;
- `src/pages/` — полноценные страницы всех основных разделов;
- `src/styles/` — дизайн-токены, глобальные, компонентные и страничные стили;
- `src/data/` — машинные данные, включая автоматически генерируемые таблицы спряжения;
- `scripts/` — импорт данных и генерация статических HTML-маршрутов;
- `content/es-419/` — содержательный пакет языка, отделённый от интерфейсного ядра.

Цветовая семантика: красный — окончания, синий — формы и множественное число, зелёный — неправильные формы, фиолетовый — паттерны, серый — грамматические комментарии.

Светлая и тёмная темы используют одну систему токенов; выбор пользователя сохраняется локально в браузере.
